# Bitwise Operators

C provides six operators for bit manipulation; these may be only applied to integral operands, that is `char`, `short`, `long`and ìnt`, whether signed or unsigned.

- `&` bitwise AND
- `|` bitwise inclusive OR
- `^` bitwise exclusive OR
- `<<` left shift
- `>>` right shift
- `~` one's complement (unary)

The bitwise AND operator is often used to mask off some sets of bits.

```c
n = n & 0177
```
sets 0 all but the low-order bits of _n_.

---
## Explanation

`n = n & 0177` performs a bitwise AND operation between `n` and `0177`.

Important: `0177` is an octal number (because of the leading 0), which equals 127 in decimal or `01111111` in binary.

The operation masks all bits except the lowest 7 bits of `n`:
- All higher-order bits (bit 7 and above) are set to 0
- The lowest 7 bits (bits 0-6) remain unchanged

Example:

```
n        = 11010110 10101011  (arbitrary value)
0177     = 00000000 01111111  (127 in decimal)
         & ─────────────────
Result   = 00000000 00101011  (only the last 7 bits remain)
```

---

The bitwise OR operator | is used to turn bits on:

```c
x = x | SET_ON;
```
sets to one in x the bits that are set to 1 in *SET_ON*

The bitwise exclusive OR operator ^ sets a one in each bit position where its operand have different bits, and zero where they are the same.

> **Important**
> One must distinguish the bitwise operators & and | from the logical 
> operators && and ||, which imply a left-to-right evaluation of a truth value.

The shift operators << and >> perform left and right shifts of their left operand by the number of bit positions given by the right operand, which must be non-negative.
Thus
```c
x << 2;
```
shifts the value of _x by two positions, filling vacated bits with 0; this is equivalent to multiplications by 4.

---
**Example**

```
x        = 00000000 00000101  (5 in decimal)
x << 2   = 00000000 00010100  (20 in decimal)
```

Shifting left by 2 positions:
- Each bit moves 2 positions to the left
- The rightmost 2 positions are filled with 0
- Result: 5 × 4 = 20

Similarly, `x >> 2` would shift right and divide by 4.


The unary operator ~ yields the one's complement of an integer; that is it converts each 1-bit into a 0-bit and vice versa.

 **Illustration**
 ```c
 /* getbits: get n bits from position p */
 unsigned getbits(unsigned x, int p, int n)
 {
    return (x >> (p+1-n) & ~(~0 << n));
 }
```

**Explanation:**

This function extracts `n` bits from `x`, starting at position `p` (counting from the right, starting at 0).

The expression works in several steps:

1. `~0` creates a value with all bits set to 1
2. `~0 << n` shifts left by n positions, creating n zeros on the right
3. `~(~0 << n)` inverts it, creating a mask with exactly n ones on the right
4. `x >> (p+1-n)` shifts x right to align the desired bits to the rightmost position
5. The `&` operation extracts only the n bits using the mask

**Example:**

Let's extract 3 bits from position 4:
```c
x = 214 (binary: 11010110)
p = 4
n = 3
```

Step by step:
```
1. ~0          = 11111111 11111111  (all 1s)
2. ~0 << 3     = 11111111 11111000  (shifted left 3)
3. ~(~0 << 3)  = 00000000 00000111  (mask with 3 ones)

4. p+1-n       = 4+1-3 = 2
5. x >> 2      = 00110101 (shifted right 2 positions)
                 11010110 → 00110101
                 
6. 00110101 & 00000111 = 00000101 (extract last 3 bits)

Result: 5 (binary: 101)
```

The bits at positions 2, 3, and 4 of the original number (which are "101") have been extracted.
